# Service Feature Roadmap

기준 문서:
- `agent/PRD.md`
- `agent/task1.md` ~ `agent/task8.md`

## 목적

현재 구현은 "번역 펫 MVP"로 핵심 동작(투명 오버레이, 번역 호출, 설정/저장, 트레이)이 완료된 상태다.  
다음 단계에서는 반복 사용성과 신뢰성을 올리는 기능을 우선 추가해야 한다.

## 우선순위 A (단기, 즉시 가치)

### 1) 확인형 클립보드 번역 (보안 기본값)
- 배경: 단축키 기반 자동 번역은 빠르지만 민감 정보 오전송 위험이 큼.
- 제안:
  - 기본값: 클립보드 자동 "제출" 금지, 입력창에만 채우기
  - 옵션: 자동 제출은 opt-in
  - 보호: 민감 패턴 감지 시 제출 차단
- 기대효과: 유출 위험 감소 + 사용자 신뢰 상승

### 2) 번역 결과 액션 강화
- 배경: 현재는 말풍선 출력 중심이라 후속 행동(복사/재사용)이 불편함.
- 제안:
  - 복사 버튼
  - 다시 번역(타겟 언어 변경 포함)
  - 길이 긴 결과는 확장 보기 모달
- 기대효과: "번역 결과를 실제 업무에 쓰는" 전환율 상승

### 3) 오류 복구 UX 개선
- 배경: API 실패 시 메시지 제공은 되지만 복구 동선이 약함.
- 제안:
  - 재시도 버튼
  - 자격증명 오류 시 설정창 바로 열기
  - 네트워크 오류/할당량 오류를 구분해 안내
- 기대효과: 실패 상황 이탈률 감소

## 우선순위 B (중기, 재방문 강화)

### 4) 로컬 히스토리 + 즐겨찾기
- 배경: 논문/자료 번역은 재조회 니즈가 큼.
- 제안:
  - 로컬 히스토리 저장 (기본 OFF, opt-in)
  - 즐겨찾기/핀 고정
  - 검색(원문/번역문)
- 기대효과: 재사용성 증가, 반복 작업 시간 단축

### 5) 용어집(Glossary) / 번역 스타일 프리셋
- 배경: 사용자별 선호 번역어가 다름.
- 제안:
  - 사용자 정의 용어집
  - "직역 우선 / 자연어 우선" 프리셋
- 기대효과: 번역 품질 체감 향상, 도메인 적합성 증가

### 6) 아카데믹 모드 (PRD Future Scope 구체화)
- 배경: PRD의 핵심 타겟(논문/해외 자료 사용자)에 직접 가치.
- 제안:
  - 문장 구조 분석
  - 직역/의역 분리
  - 핵심 용어 정리
  - 한 줄 요약
- 기대효과: 단순 번역 툴에서 "학습/리서치 보조 도구"로 포지셔닝 확대

## 우선순위 C (확장)

### 7) 화면 선택 번역(OCR)
- 배경: 복사 불가능한 PDF/이미지 텍스트 번역 수요 대응.
- 기대효과: 사용 가능 시나리오 확장

### 8) 캐릭터 개인화
- 제안:
  - 스킨/테마
  - 자율 멘트 빈도 조절
  - 방해 금지 모드
- 기대효과: 장기 사용 피로도 감소, 개인화 경험 강화

## 추천 실행 순서 (이슈 분할)

1. `Clipboard UX Hardening`: 확인형 제출 + 민감 패턴 차단
2. `Result Actions`: 복사/다시번역/확장보기
3. `Recovery UX`: 재시도 + 설정창 딥링크
4. `Local History`: opt-in 기록/검색/즐겨찾기
5. `Glossary & Presets`
6. `Academic Mode`

## 수용 기준 (문서 기준)

- 각 기능은 보안 기본값(최소 권한, 민감정보 보호)을 우선한다.
- 새 기능은 `main.js`(권한/보안), `preload.js`(IPC 경계), `renderer.js`(UX)를 분리해 설계한다.
- 작업 단위는 이슈 1개 + 브랜치 1개 + PR 1개로 진행한다.
