# Task 8. 통합 검증, 안정화, 배포 준비

## 목표
핵심 시나리오를 통합 테스트하고, 릴리스 가능한 수준으로 안정화한다.

## 범위
- E2E 수동 테스트 시나리오 정리:
  - 단축키 번역
  - 클립보드 번역
  - 클릭 스루
  - 드래그 이동
  - 트레이 설정 저장/재실행 복원
- 오류/성능 점검:
  - API 실패 복구
  - 메모리 누수/과도한 타이머 점검
  - 창 포커스/최상단 유지 동작 검증
- 로그 레벨/디버그 플래그 정리.
- 패키징 설정 초안 (Windows 우선, 필요 시 macOS/Linux 확장 포인트 명시).
- 운영 문서 작성:
  - 설치/실행
  - 환경변수/키 설정
  - 알려진 제한사항

## 산출물
- QA 체크리스트 및 결과.
- 배포 가능한 빌드 설정 초안.

## 완료 기준 (DoD)
- PRD 핵심 기능(번역/캐릭터/투명창/설정)이 끊김 없이 동작.
- 치명 오류 재현 케이스가 없고, 알려진 이슈 목록이 문서화됨.
- 신규 개발자가 문서만으로 로컬 실행 가능.
